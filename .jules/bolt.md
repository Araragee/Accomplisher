- React Fast Refresh (`react-refresh/only-export-components`) strictly prefers files to export either only components or only functions, not both. We handled this by separating the `<Link>` component from `src/router.ts`.
- It's a standard pattern to export a Context Provider and its custom hook from the same file. To avoid Fast Refresh warnings here, disable the rule via `/* eslint-disable react-refresh/only-export-components */` at the top of the file.
- The `react-hooks/set-state-in-effect` rule is a custom/experimental warning that triggers when calling setState synchronously within an effect body (e.g., calling `setLoading(true)` at the start of a data fetching effect). In our cases, this was the intended architecture and it was disabled line-by-line via `// eslint-disable-next-line react-hooks/set-state-in-effect`.
